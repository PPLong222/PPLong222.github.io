Linux指令

Linux指令

ls

  • -i 查看inode号
  • -ll size+权限
  • -lh 同上，以不同存储单位显示

stat

  • filename 查看某个文件的inode信息

System

fh

磁盘相关信息

  • -h：查看电脑上挂载的分区的大小

Process

ps

  • -A ：所有进程
  • -a ：同一终端下的所有进程
  • -u ：显示指定用户的所有进程
  • ps e ：显示每个程序使用的环境变量
  • ps f： 使用ASCII字符显示树状结构
  • -e ： 与-A指令一样
  • -x ：显示所有程序，不按照终端机区分
  • -au：显示详细信息
  • -aux：显示所有包含其他使用者的行程
    • USER: 行程拥有者
    • PID: pid
    • %CPU: 占用的 CPU 使用率
    • %MEM: 占用的记忆体使用率
    • VSZ: 占用的虚拟记忆体大小
    • RSS: 占用的记忆体大小
    • TTY: 终端的次要装置号码 (minor device number of tty)
    • STAT: 该行程的状态:
      • D: 不可中断的静止
      • R: 正在执行中
      • S: 静止状态
      • T: 暂停执行
      • Z: 不存在但暂时无法消除
      • W: 没有足够的记忆体分页可分配
      • <: 高优先序的行程
      • N: 低优先序的行程
      • L: 有些页被锁进内存
      • l 多进程的
      • + 位于后台的进程组；
    • START: 行程开始时间
    • TIME: 执行的时间
    • COMMAND:所执行的指令
  • -ef ：显示所有进程信息，包括命令行（与grep 连用来查找特定进程）
  • -eo：自定义显示，后面跟参数名,

ps工具标识进程的5种状态码:
D 不可中断 uninterruptible sleep (usually IO)
R 运行 runnable (on run queue)
S 中断 sleeping
T 停止 traced or stopped
Z 僵死 a defunct (”zombie”) process (进程已终止, 但进程描述符存在, 直到父进程调用wait4()系统调用后释放)

pstree -p：以树结构显示进程
pstree -apnh ：显示进程间的关系

kill

  • kill ，系统发送sigterm信号给程序，程序受到后可能立即停止、可能释放相应资源后停止、可能继续运行
  • kill -9，系统发送的是SIGKILL，即exit信号，不会被系统阻塞，直接杀掉程序
  • kill -15，发送信号给程序，程序根据自己情况选择停止时间（程序在等待IO，就不能立马停止）
  • -CONT 继续暂停的进程
  • -STOP 暂停进程

&

将进程放在后台运行，并返回一个id号。但关闭当前会话或终端后，就会停止执行

nohup

让进程在后台执行，并且关闭当前终端，进程也会继续执行，适合服务器进行的操作。但关闭终端后，在另一个终端里看不到jobs -l当前的信息。并且输出在终端的信息会显示在当前目录下的nohup.txt中

fg\bg\jobs

  • fg pid：将一个在后台暂停的命令，变成在前台继续执行
  • bg pid：将一个在后台暂停的命令，变成在后台继续执行。
    • 但是例如进程有echo输出时，会输出到当前的终端。
  • jobs：显示当前有多少在后台运行的命令（仅在当前终端启动的）
    • -l ：显示pid

ctrl+z：将一个正在前台执行的命令放到后台，并使其暂停

IO

echo

  • -e 能识别转义字符（\n等）

printf

File

chmod

change mode: 控制用户对文件的命令
Linux的文件调用权限分为三级：Owner \ Group \ Other User

r-w-x : Read-Write-Execute

只有文件所有者Owner和超级用户可以修改可以修改文件和目录的权限，也能够使用八进制修改权限

  • u：表示文件的拥有者
  • g：表示与该文件拥有者所属的同一个群体
  • o：表示其他人
  • a：表示三者都是
  • +：增加权限 - 取消权限 = 唯一设定权限
  • -R：对目录下的文件与子目录递归进行操作的

但是有一个奇怪的现象，我把一个文件设置为不可读不可写，然后我对文件强制进行修改 wq!保存，然后恢复权限去查看，发现数据都改变为强制修改的数据了

mv

cp

find

  • -name “filename” ：" * xxxx * "
    • ! -name 找到文件名字不是xxx结尾的

Network

ssh

scp

netstat

  • -l ：显示监听端口
  • -a：显示所有端口

ifconfig

显示和配置网卡的网络参数，可以设置MTU。

route

显示并设置静态路由表

  • -n：不执行DNS反向查找，直接显示数字形式的IP地址
  • -e：netstat格式显示路由表
  • -v：详细信息显示

删除路由记录
sudo route del default / (or xxxxx)

增加默认路由记录（0.0.0.0）
sudo route add default gw 172.20.10.1 window 2796 dev wlo1
