Java8特性学习

Java8特性学习

Mybatis中出现了一些与Lambda相关的函数方法，使得在编程时非常的方便，但自己却不知道如何使用基于Java8的语法

概述

优点

  • 大数量下处理集合效率高
  • 代码可读性高，代码简洁，开发快速
  • 易于并发编程
  • 减少嵌套

函数式编程思想

面向对象的思想关注的说用什么对象完成什么事情，而函数式编程关注是对数据进行了什么操作

Lambda表达式

本质是JDK8 的语法糖，能对某些匿名内部类的写法进行简化，是函数式编程思想的重要体现。不关注是什么对象，而关注我们对数据进行了什么操作

示例

其中一种较为常见的Lambda表达式如下，传参是一个接口并且接口中只定义了一个方法

new Thread(new Runnable() {
            @Override
            public void run() {
                System.out.println(" Runnable Thread Start");
            }
        }).start();
new Thread(()-> System.out.println()).start();

注意，这里()代表的那个接口中的唯一的那个方法，而不是指代那个接口

返回值

public class LambdaTest {
    public static void main(String[] args) {
        testFunc(value ->  value);
    }

    public static void testFunc(MyInt myInt) {
        String value = myInt.func("hello");
        System.out.println(value);
    }
}

interface MyInt{
    String func(String value);
}

如果匿名类函数只有一行，则可以直接不加大括号和return，也表示返回值。否则则需要添加大括号和return（如果有返回类型要求的话）

优化规则

  • 函数体只有一行代码时，可以忽略掉大括号，return语句以及随后的分号
  • 方法只有一个参数时小括号可以省略
  • 参数类型可以省略

Stream流

用来对集合或数组进行链状流式的操作

常用操作

创建流

集合对象.stream()

中间操作

filter：根据某个条件过滤，返回值为Boolean值

map：通过某种转化，把流转化为承载另一种数据（格式）的流，只能把一个对象转化成另一个对象来作为流中的元素

sort：有参和无参两种，有参需要提供Comparator

distinct： 去重操作，通过类的内部equals方法进行比较

limit skip：限制总条数为多少或跳过开头的n个元素

flatMap：把一个对象转化为多个对象作为流中的元素，比如一个对象中有一个List<Object> 则flatMap可以将所有对象的该属性整合成一个承载Object的流，而map只能转化为承载List<Object>的流

终结操作

惰性求值：如果一个流中没有终结操作，则这个流不会启动
一次性：一个流进过一个终结操作后则不能再次被使用

forEach

count

max&min: 需要传入Optional<Compartor>，指出以哪一种方式获得最大最小值

collect

把当前流转化为一个集合

// List 转化  stream.collect(Collectors.toList());

Map<String, Integer> collect = defaultPersonArrays.stream()
                .collect(Collectors.toMap(person -> person.getName(), person -> person.getAge()));

这里Collectors是java.util.stream下的类

anyMatch: 判断有任意符合匹配条件的元素，只要有一个则就满足

allMatch: 全部满足为true

noneMatch: 流中的元素是否都不符合

findAny: 获取流中对任意一个元素，无法保证一定是流中的第一个元素

findFirst: 获取流中的第一个元素

reduce

对流中的数据按照制定的计算方式给出一个结果（缩紧操作）。可以直接给出初值也可以不设置初值（不设置时第一个元素为初值）

Optional

优雅地避免空指针异常，减少非空判断对代码行数。每次通过查询或计算得到结果时，若方法内部没有处理空数据的逻辑，则在外部代码部分就必须判断结果是否为空

使用

类似包装类

将一个对象封装成Optional

Optional<Person> person1 = Optional.ofNullable(person);
// 传入对象必须非空
Optional<Person> person1 = Optional.of(person);

ifPresent

安全消费，如果不为空的话，则进行处理

orElseGet

安全获取值，如果数据不为空则返回该数据，为空则根据代码返回另一个值

orElseThrow

安全获取值，如果值不存在则抛出一个自己指定的异常

get 不安全获取值，如果Optional中不存在值，则会抛出异常

filter 如果不符合条件，则会变成一个无数据的Optional对象

isPresent 是否存在值 返回boolean

map&flatMap 同Stream，只是变成了单一对象

函数式接口

函数接口： 只有一个抽象方法的接口（默认方法不算）

**JDK函数式接口都加上了@FunctionalInterface，但无论是否有此注解，只要接口中只有一个方法则都是函数式接口。此注解只是一个标注 **

JDK中默认给出了几个函数式接口。Consumer消费接口、 Function计算转换接口（接受一个T类型数据并返回为R类型数据）、 Predict判断接口、生产型接口Supply（生成一个类型的对象）等

至于BiConsumer 、 BiFunction等，差别是需要两个参数

默认方法搭配

例如Predict接口中还有and、or之类的默认方法，在使用stream时可以通过or、and来拼接条件（但似乎无法使用lambda来实现拼接，只能使用匿名内部类的方式）

方法引用

使用lambda时若方法体中只有一个方法的引用，则可以用方法引用进一步优化代码，例如Mybatis中的lambdaQueryWrapper使用查询时就有Class::Function

格式：类名/对象名 :: 方法名

  • 调用静态方法：抽象方法只有一行嗲吗，且其中的参数都按照顺序传入静态方法中中，额可以直接使用 类名:方法名 的方式（参数都不用写了）

  • 引用对象的实例方法：前置条件与上类似，但最终为 对象名:方法名

  • 引用类的实例方法： 实现的方法体只有一行代码，且这行代码调用了第一个参数的成员方法，并且实现的接口的方法中剩余的所有参数都按照顺序传入到这个成员方法中。类名:方法名

  • 构造器引用：一样，抽象方法中定义的参数都按照顺序传入了构造器中，类名:new
